# Story 1.3 — Migrar Frontend para Azure Web App

> **Epic:** [EPIC-001 — VM-to-WebApp Modernization](../epics/EPIC-001-vm-to-webapp-modernization.md)
> **Status:** Draft
> **Estimated:** 30 min
> **Pré-requisito:** [Story 1.2](1.2.story.md) concluída (backend em PaaS, frontend e DB ainda em VM)

---

## Story

**Como** aluno do evento "Copa do Mundo Azure",
**eu quero** substituir a `vm-front` por um **Azure Web App for Windows** (`fifa2026-web`),
**para que** o frontend também passe a ser PaaS — eliminando outra VM, ganhando HTTPS automático com domínio `azurewebsites.net` e descobrindo na prática como o **mesmo `web.config`** das VMs roda sem mudanças no Web App, porque a Microsoft usa o mesmo IIS por baixo.

---

## Contexto

Estado **antes**:
```
Internet ──▶ vm-front (IIS) ──▶ fifa2026-back (Web App) ──▶ vm-db (SQL Server)
```

Estado **depois**:
```
Internet ──▶ fifa2026-web (Azure Web App) ──▶ fifa2026-back (Web App) ──▶ vm-db (SQL Server)
```

A `vm-front` pode ser desligada. Restou só a `vm-db` para a Story 1.4.

**Insight chave:** o `web.config` que rodava na `vm-front` **funciona idêntico** no Azure Web App, porque ambos usam IIS. Só muda 1 linha (a URL do backend).

---

## Acceptance Criteria

1. **AC-1** — Azure Web App `fifa2026-web` provisionada (Windows, Node 20, mesmo plano `fifa2026-plan`).
2. **AC-2** — Build do frontend gerado **com BACKEND_URL apontando para `https://fifa2026-back.azurewebsites.net`**.
3. **AC-3** — Conteúdo do `dist/` (incluindo `web.config` com placeholder substituído) deployado em `fifa2026-web`.
4. **AC-4** — `dist/web.config` tem o `Rewrite url=` apontando corretamente para a URL do Web App backend (verificável via Kudu / `https://fifa2026-web.scm.azurewebsites.net/wwwroot/web.config`).
5. **AC-5** — App Setting `FRONTEND_URL` no `fifa2026-back` atualizada para `https://fifa2026-web.azurewebsites.net` (CORS allowlist).
6. **AC-6** — **Smoke test passa:** `https://fifa2026-web.azurewebsites.net` carrega a SPA, login funciona, listar jogos funciona.
7. **AC-7** — `vm-front` pode ser desligada sem afetar a app.

## Dev Notes

**Por que o build precisa saber o BACKEND_URL antes de ser feito?**
- O `web.config` do IIS **não suporta env vars** em rewrite rules — o valor precisa ser literal
- Por isso o template em `public/web.config` tem o placeholder `__BACKEND_URL__`, e o postbuild script (`scripts/set-backend-url.mjs`) substitui pelo valor real
- Em qualquer mudança da URL do backend, **rebuild necessário**

**Por que mantemos `vm-db` em VM?**
- Cada story muda **uma** camada
- Próxima story (1.4) migra o SQL Server para Azure SQL
- Você vai aprender que migrar dados (DB) é fundamentalmente diferente de migrar compute (Web App) — há contratos de schema, dados em produção, downtime

**Custo desta story (acumulado com 1.2):**
- 2 Web Apps no mesmo plano B1 = ainda ~$13/mês (plano B1 acomoda múltiplos apps)
- Ainda paga 2 VMs (vm-front e vm-db) se não desligou

## Validation / Smoke Test

```bash
# 1. SPA carrega
curl -I https://fifa2026-web.azurewebsites.net/
# HTTP/1.1 200 OK

# 2. Proxy /api funciona (chama o web.config -> backend)
curl https://fifa2026-web.azurewebsites.net/api/health
# {"status":"ok","timestamp":"..."}

# 3. Conferir que o web.config deployado tem URL correta (via Kudu)
curl -u "<usuário>:<senha-do-publish-profile>" \
  https://fifa2026-web.scm.azurewebsites.net/api/vfs/site/wwwroot/web.config | grep "Rewrite url"
# <action type="Rewrite" url="https://fifa2026-back.azurewebsites.net/api/{R:1}" />

# 4. Fluxo end-to-end
TOKEN=$(curl -s -X POST https://fifa2026-web.azurewebsites.net/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@fifa2026.com","password":"admin123"}' | jq -r '.token')
curl -s https://fifa2026-web.azurewebsites.net/api/matches \
  -H "Authorization: Bearer $TOKEN" | jq '.matches | length'
# 104
```

## Rollback

Se algo der errado:

1. Reapontar DNS / informar audiência para usar `http://<IP_PUBLICO_FRONT>` (vm-front) novamente
2. Reverter `FRONTEND_URL` no backend:
   ```bash
   az webapp config appsettings set -g fifa2026-paas-rg -n fifa2026-back \
     --settings FRONTEND_URL="http://<IP_PUBLICO_FRONT>"
   ```
3. (Opcional) Apagar `fifa2026-web`: `az webapp delete -g fifa2026-paas-rg -n fifa2026-web`

App volta ao estado da Story 1.2.
